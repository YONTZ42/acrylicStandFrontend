


アクリルスタンド（アクスタ）のシミュレーションから購入・共有までを促すアプリ、非常に面白くポテンシャルの高いプロダクトですね！「推し活」をしているユーザー（主にZ世代〜ミレニアル層）をターゲットとする場合、現在の**「機能ベースの画面分割（Studio / Collection / Galleries）」は少し事務的で、ユーザーの熱量（エモさ）を削いでしまう可能性**があります。

仕様書にとらわれず、ユーザーが「推しを可愛く作りたい！」「最高の祭壇を作って自慢したい！」「手元に実物が欲しい！」と感情を動かされるような、**最適化したユーザーフローとポップでスタイリッシュなUI/UXデザイン**をご提案します。

---

### 1. ユーザーフローとURL設計の再構築

現状のフロー（Studioで作る → Collectionで管理 → Galleriesで3D配置）は、工程が多く「システム的」です。推し活ユーザーにとっては**「アクスタを錬成する（作る）」**と**「祭壇に飾る（並べる）」**の2つの体験がコアです。

#### 💡 提案する新しいユーザーフロー
**「Collection（一覧）」と「Gallery（3D配置）」を統合し、『祭壇（Room / Gallery）』として1つのメイン画面にします。**

* **`/` (LP 兼 クイック錬成)**
  * ファーストビューで動く3Dアクスタを見せ、「推しをアクスタにしてみる」ボタンを配置。
  * 1枚画像をアップロードさせ、裏で `Guest ID` を発行しつつ、サクッと背景切り抜き＆アクスタ化を体験させる。「祭壇に飾る」ボタンでアプリ本体へ。
* **`/app/room` (メイン画面：祭壇 兼 コレクション)**
  * **画面の大部分は3Dの祭壇（Gallery）。**
  * **画面下部にドロワー（引き出し）形式で「所持しているアクスタ一覧（旧Collection）」**があり、そこからドラッグ＆ドロップで直感的に祭壇に飾れるUI。
  * 飾ったアクスタをタップすると、「位置調整」や**「カートに入れる（物理購入）」**のポップアップが出る。
* **`/app/studio` (アクスタ錬成所)**
  * 新しいアクスタを作る専用画面。画像アップロード、AI背景切り抜き、エフェクト付与（アニメ風、ピクセルアート風）、台座のカスタマイズを行う。
* **`/g/{slug}` (共有ビューア)**
  * 友達に見せるための「〇〇の祭壇」URL。見た人が「自分も作りたい！」と思える導線（CTA）を強く配置。

### 2. 「推し活向け」UI/UXデザインコンセプト

ただの「ツール」ではなく、「おもちゃ箱」や「魔法のアイテム」を触っているような体験（**トイボックス・ポップ / Y2K・サイバーパステル**）を目指します。

#### ① カラースキーム＆質感（マテリアル）
* **アクリル＆グラスモーフィズム**
  * アクスタの「透明感」「アクリル感」をUI自体にも取り入れます。半透明の背景（`backdrop-filter: blur`）に、白い細いボーダー（`border-white/20`）を合わせたガラス風のパネルデザイン。
* **ホログラム＆オーロラ（推し色）**
  * アプリのアクセントカラーは固定せず、グラデーション（オーロラカラー）を使用。
  * ボタンのホバー時や、アクスタ完成時に「キラッ」と光るホログラムエフェクト（CSSアニメーション）を入れる。

#### ② タイポグラフィと形状
* **丸みと親しみやすさ（Bouncy & Rounded）**
  * ボタンやカードは角丸（`rounded-2xl` やフルピル型 `rounded-full`）を多用。
  * トランジションには弾むようなイージング（`cubic-bezier(0.68, -0.55, 0.26, 1.55)`）を使い、ポップな動きをつける。

#### ③ 専門用語を「エモい」ワーディングに変換
* `YOLO` や `rembg` といった技術用語は裏側に隠します。
* 「背景を透過する」→ **「魔法で切り抜く ✨」**
* 「エフェクトを適用」→ **「エモいフィルターをかける 🎨」**
* 「Galleryに保存」→ **「祭壇に飾る 💖」**

### 3. 主要画面・コンポーネントの具体的デザイン設計

#### A. Studio画面（アクスタ錬成所）
画像処理（切り抜き・AI生成）という**待機時間が発生する部分をエンタメ化**します。
* **レイアウト:** 中央に大きくプレビュー領域。下部または右側に「スタイル選択」のサムネイルを横スクロールで配置（Instagramのフィルター選択のような直感的なUI）。
* **ローディングUI:** 「画像を処理中...」という文字ではなく、魔法陣が回るアニメーションや、キラキラしたパーティクルが舞う演出にし、「可愛いものが生まれるワクワク感」を演出します。
* **AI生成のプレビュー:** 元画像と生成後の画像をスワイプ（Before/Afterスライダー）で比較できると「すごさ」が伝わります。

#### B. Room画面（祭壇・3Dギャラリー）の革新
一番の滞在場所であり、購買意欲を高める画面です。
* **シームレスなUI構造:**
  ```text
  [ 上部 80% ]: 3Dキャンバス（PlayCanvas）。グリグリ回せる。[ 底部 20% ]: 半透明のグラスモーフィズムな引き出し（ドロワー）。
                ここに追加済みのアクスタ（2Dアイコン）が並ぶ。
  ```
* **インタラクション:**
  底部のドロワーからアクスタを指でスッと3D空間にドラッグすると、ポンッ！というエフェクトとともに3D上に配置される。
* **購入への導線（最重要）:**
  3D空間で配置したアクスタをタップすると、そのアクスタの周りにふんわりとメニューが浮かび上がります。
  一番目立つカラー（オーロラグラデーション等）で **「🛒 実物を購入する（¥1,500〜）」** ボタンを配置。「自分が作った完璧な配置」を見た直後なので、購買率が格段に跳ね上がります。

#### C. 物理購入を促す「シミュレーションの極致」
単に3Dモデルを見るだけでなく、**「現実世界でのスケール感」**を感じさせます。
* 3Dギャラリーの背景を「一般的な学習机」「PCモニターの前」「カフェのテーブル」などに切り替えられるようにし、「自分の生活空間に推しのアクスタがあるイメージ」を強く抱かせます。
* 共有（`/g/slug`）された側も、他人の祭壇を見て「このクオリティのアクスタが買えるんだ」と気づけるよう、共有画面右上に常に「自分もアクスタを作る・買う」ボタンを追従させます。

### 実装に向けたフロントエンド構成のアドバイス

現在のディレクトリ構成をこのフローに合わせるなら、以下のように整理することをおすすめします。

```text
src/app/
  routes/
    app/
      room/            ← (旧 galleries + collection) メインの祭壇画面。3D+ドロワー
      studio/          ← 画像アップロード・AI生成・切り抜き
      shop/            ← カート・決済画面（将来拡張）
    viewer/
      PublicRoomPage.tsx ← 共有された祭壇ビューア
```

**必要なCSS技術・ライブラリの提案:**
1. **Tailwind CSS**: 現状通り使用。`tailwind.config.js` で `colors: { glass: 'rgba(255, 255, 255, 0.2)' }` などを定義しグラスモーフィズムを使いやすくする。
2. **Framer Motion (React用)**: ドロワーの引き上げ、ポップアップのバウンス、画面遷移のシームレスなアニメーションを簡単かつ高品質に実装するために必須レベルでおすすめです。
3. **パーティクルエフェクト**: `tsparticles` や `canvas-confetti` を用いて、アクスタ完成時や購入ボタンタップ時に画面内にキラキラを降らせる。
